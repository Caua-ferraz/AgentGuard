# Support

AgentGuard offers two support tiers: best-effort community support for the
open-source project, and commercial support via AgentGuard Cloud.

## Open-source support

The AgentGuard OSS project is supported on a best-effort basis by the
maintainers and the wider community.

- **Bug reports and feature requests:** please open a GitHub Issue using
  the provided issue templates. Search existing issues first.
- **Questions and usage help:** GitHub Issues are also acceptable for
  setup and integration questions; please tag them clearly.
- **No SLA:** we do not commit to a response time on the OSS project,
  although we typically respond within a few business days.
- **Security reports:** see [`SECURITY.md`](SECURITY.md). Do **not**
  open a public issue for vulnerabilities.

## AgentGuard Cloud support

AgentGuard Cloud is a managed offering with commercial support. Pricing,
SLAs, and regional availability are still being finalized.

- **Waitlist / early access:** <https://agentguard.lictorate.com>
- **Sales / procurement enquiries:** cauaferraz@lictorate.com
- **SLA, pricing:** TBA. Cloud customers receive a dedicated response
  channel and an incident-response SLA defined in their order form.

If you are evaluating AgentGuard for a regulated or large-scale deployment
and need procurement artefacts (vendor questionnaires, SOC 2 status, DPIA
inputs), please reach out via the waitlist link above.
