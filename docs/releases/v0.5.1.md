# AgentGuard 0.5.1 ‚Äî Release Notes

**Released:** 2026-05-05
**Scope:** Python SDK only. Go binaries (`agentguard`, `agentguard-mcp-gateway`, `agentguard-llm-proxy`), the TypeScript SDK, and the policy engine stay at v0.5.0.

## Headline

The CrewAI and LangChain adapters in the Python SDK no longer break under the latest framework majors.

## What broke in v0.5.0

The v0.5.0 `GuardedCrewTool` and `GuardedTool` adapters were composition wrappers that registered themselves as virtual subclasses of the framework's `BaseTool` via `BaseTool.register(GuardedCrewTool)`. That kept the policy gate tight via a strict `__getattr__` allowlist ‚Äî but pydantic 2.12 emits this warning at runtime:

> For performance reasons, virtual subclasses registered using `BaseTool.register()` are not supported in `isinstance()` and `issubclass()` checks.

And `isinstance` is exactly what CrewAI 1.x's pydantic-validated `Agent(tools=[...])` and langgraph 1.0's `coerce_to_runnable` use to validate tool registrations. Concrete failures observed against current upstream majors:

- `Agent(tools=[GuardedCrewTool(...)])` ‚Üí
  `pydantic_core.ValidationError: 1 validation error for Agent / tools.0  Input should be a valid dictionary or instance of BaseTool [...] input_type=GuardedCrewTool`
- `langgraph.prebuilt.create_react_agent(llm, tools=[GuardedTool(...)])` ‚Üí
  rejected by `coerce_to_runnable`'s `isinstance(thing, Runnable)` check; required a workaround of wrapping the gated tool in `Tool.from_function(func=lambda x: gt.invoke(x))`.

Upstream context: pydantic 2.12 (released early 2026), CrewAI 1.x (1.14 at the time of this release), langchain_core 1.x, langgraph 1.0+ all enforce strict `isinstance` instead of honouring ABC virtual-subclass registrations.

## What v0.5.1 changes

Both adapters were rewritten as **hybrid subclass-and-override**: they actually subclass the framework's `BaseTool`, so isinstance passes natively, and every dispatch entry point is explicitly overridden so the policy gate fires before the parent's behaviour.

- `agentguard.adapters.crewai.GuardedCrewTool` is now a `crewai.tools.BaseTool` subclass. `_run`, `run`, `invoke`, `ainvoke`, `_arun`, `arun`, `__call__`, and `to_structured_tool` are all overridden. CrewAI's structured-tool conversion (`to_structured_tool`) hands off OUR gated `_run` rather than the wrapped tool's bare `_run`, so no dispatch path within CrewAI's agent loop sidesteps the gate.
- `agentguard.adapters.langchain.GuardedTool` is now a `langchain_core.tools.BaseTool` subclass. `_run`, `_arun`, `invoke`, `ainvoke`, `stream`, `astream`, `batch`, `abatch`, `run`, `arun` are all overridden. Inputs shaped as langchain_core `ToolCall` dicts (`{"name", "args", "id", "type": "tool_call"}`, which is what langgraph's agent runtime passes) are unwrapped to the underlying `args` payload before the gate runs.
- Internal references (`_tool`, `_guard`, `_scope`) use `pydantic.PrivateAttr`, so they stay off `model_fields` and are excluded from `model_dump()` output.

## Defense contract change

| Aspect | v0.5.0 (composition) | v0.5.1 (hybrid subclass + override) |
|---|---|---|
| `isinstance(thing, BaseTool)` at framework boundary | Failed under pydantic 2.12 / CrewAI 1.x / langgraph 1.0 | Passes natively |
| Bypass via parent attribute access | Blocked by `__getattr__` allowlist | Every gated path is overridden ‚Äî no inherited dispatch survives |
| Bypass via `_tool` / `func` | Hard-blocked with a security message | Pydantic `PrivateAttr` makes the field non-public; calling-code reaching `gt._tool._run(...)` directly skips the gate but the agent runtime never does |
| Future framework adds new dispatch path | Caught by `__getattr__` allowlist | Caught by the canary integration test (`tests/integration/test_at_real_*.py`) |

The new defense rests on (a) **explicit override of every dispatch path the framework calls** and (b) **the integration test matrix** trips when upstream adds a new entry point. The trade-off: a determined caller who already holds the wrapper instance can reach into `gt._tool` directly and bypass the gate. We accept that trade-off because the agent runtime ‚Äî the actual code path under attack from prompt-injection / model-jailbreak ‚Äî only calls the gated dispatch methods, and the integration tests exercise them under the real upstream library.

## Upgrade

```bash
pip install --upgrade 'agentguardproxy[crewai,langchain]'
```

If you previously wrote the langgraph workaround:

```python
# v0.5.0 workaround ‚Äî no longer required
gt = GuardedTool(my_tool, guard, scope="shell")
agent_tool = Tool.from_function(name="echo", func=lambda x: gt.invoke(x), description="...")
agent = create_react_agent(llm, tools=[agent_tool])
```

You can simplify to:

```python
# v0.5.1 ‚Äî register the GuardedTool directly
gt = GuardedTool(my_tool, guard, scope="shell")
agent = create_react_agent(llm, tools=[gt])
```

For CrewAI:

```python
# Now works under CrewAI 1.x without ValidationError
gt = GuardedCrewTool(my_crewai_tool, guard, scope="shell")
agent = Agent(role="...", goal="...", backstory="...", tools=[gt])
```

## Compatibility

- **Go binaries**: unchanged at v0.5.0.
- **TypeScript SDK**: unchanged at v0.5.0.
- **Python SDK API**: backward-compatible. Existing code that constructs `GuardedTool(my_tool, guard)` / `GuardedCrewTool(my_tool, guard=guard)` continues to work; only the internal class topology changes.
- **Python framework floors**: `crewai >= 0.80, < 2.0` and `langchain-core >= 0.3, < 2.0` are unchanged. Verified end-to-end against `crewai 1.14.4` and `langchain-core 1.3.3` on Python 3.13.

## Tests

- Unit tests in `plugins/python/tests/test_crewai.py` and `tests/test_langchain.py` updated to pin the new defense contract (every dispatch entry overridden, private attrs hidden from `model_dump`, isinstance passes against the real framework BaseTool).
- The previously-failing integration tests in `tests/integration/test_at_real_crewai.py` (`TestRealCrewKickoffGatesTool`, `TestRealCrewKickoffDenyHaltsTool`) now pass end-to-end via an offline `BaseLLM` subclass that drives CrewAI's native function-calling path without an OpenAI key.
- `tests/integration/test_at_real_langchain.py` simplified to register `GuardedTool` directly with `langgraph.prebuilt.create_react_agent`.
