# AgentGuard v0.7.0 — operator summary

> 2026-06-12. The consistency + durability release. Full change list:
> [`CHANGELOG.md`](../../CHANGELOG.md) § 0.7.0. Note: **v0.6.0 was never
> tagged** — the persistent multi-tenant state milestone ships in this
> release. If you are upgrading from v0.5.x, read the v0.6 sections of
> the changelog too.

## TL;DR for upgrades

- **`serve` is stateful by default.** First run creates `agentguard.db`
  (SQLite, WAL) in the working directory; approvals, rate-limit buckets,
  and cost accumulators now survive restarts. `--persist=false` restores
  the old pure-in-memory behaviour. The store is never on the `/v1/check`
  hot path (measured p99 0.53 ms with persistence on).
- **Verdicts are now consistent across integration paths.** If your
  policies key on the filesystem `action` field, expect new (correct)
  matches through CrewAI and the MCP paths — those adapters previously
  failed to send `action` in cases where LangChain/the LLM proxy did.
  Review deny rules before upgrading agents in production.
- **Two new rule strings for dashboards/alerting:**
  `deny:gateway:invalid_response` and `deny:llm_api_proxy:invalid_response`
  (unrecognised `/v1/check` decision payloads now hard-deny with a stable
  rule on both proxies).
- **Wire format unchanged.** `schema_version: "v1"`, the `/v1/...` and
  `/v1/t/{tenant}/...` route families, and the audit JSONL schema are all
  backward-compatible. Go-level API changes (pkg/metrics accessor
  functions replacing exported vars) only affect direct library importers.

## New operator surface

| Flag / feature | Binary | What it does |
|---|---|---|
| `--fail-audit-log` (default `agentguard-fail-audit.jsonl`) | both proxies | In `--fail-mode fail-closed-with-audit`, every denial made while the central server is unreachable is appended locally as canonical `audit.Entry` JSONL — the outage window stays reconstructable. |
| `--notify-spool <path>` | `agentguard serve` | Notification events that overflow the dispatch queue spill to a JSONL spool and are redelivered by a recovery loop (including leftovers from a previous process) instead of being dropped. |
| `agentguard check --watch <file>` | CLI | Follow a JSONL file (tail -f) and verdict each appended request with one policy load for the whole stream; policy hot-reloads on edit; SIGINT returns the aggregate exit code. |
| `/v1/health` `"degraded"` status | server | Flips when audit entries are parked in the buffered overflow backlog; warnings carry corrupt-line and notify-drop counts. HTTP stays 200 — act on the `status` field. |
| `tools.listChanged: true` | MCP gateway | Upstream `notifications/tools/list_changed` frames are forwarded to the host; clients re-pull `tools/list` when an upstream's tool set changes. |
| `agentguard tenant put\|list\|rm` | CLI | Register per-tenant policies (requires `--persist`); each tenant gets isolated approvals, limits, costs, and tenant-scoped audit reads. |

## New Prometheus series

- `agentguard_audit_buffered_dropped_to_overflow_total` /
  `agentguard_audit_buffered_drained_from_overflow_total` /
  `agentguard_audit_buffered_queue_depth` — buffered-audit durability.
  Alert when dropped − drained stays > 0 (that backlog also drives the
  health `"degraded"` status).
- `agentguard_notify_spooled_to_disk_total` /
  `agentguard_notify_despooled_total` — notify-spool durability.
- The fail-mode rule strings above are greppable in audit logs and
  dashboards.

## SDK notes

- Python: LangChain `stream`/`astream` re-validate the policy decision
  every 10 s (`agentguard.adapters.langchain.STREAM_REGATE_SECONDS`), so
  revoking a permission mid-stream cuts the stream off with the same
  typed `PermissionError`. Imports are unchanged; the package was split
  internally into `core.py`/`decorators.py` with full re-exports.
- The Python MCP adapter now maps `create*` tools to the `write` action;
  CrewAI now sends the inferred filesystem `action` (see the verdict-
  consistency note above).
