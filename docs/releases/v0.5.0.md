## Headline changes

### Two new wire-level proxy binaries

**`agentguard-mcp-gateway`** â€” a JSON-RPC bridge for MCP-aware clients. Spawns one or more downstream MCP servers as subprocesses, aggregates their tool inventories with namespace prefixing (e.g. `fs:read_file`, `github:create_issue`), and gates every `tools/call` through the AgentGuard policy engine before forwarding to the upstream. Five client integration configs ship in `examples/`: Claude Desktop, Cursor, Cline, Continue, Zed.

**`agentguard-llm-proxy`** â€” an HTTP server speaking OpenAI Chat Completions and Anthropic Messages wire formats. Set `OPENAI_BASE_URL=http://127.0.0.1:8081/v1` (or `ANTHROPIC_BASE_URL=http://127.0.0.1:8081`) and existing SDK code flows through. Tool calls inside response streams are buffered, gated, and either flushed byte-identical (ALLOW), rewritten to a synthetic refusal (DENY), or surfaced for human approval (REQUIRE_APPROVAL) â€” all without SDK changes. Both streaming and non-streaming responses are gated.

Quickstart docs: [`docs/QUICKSTART_MCP.md`](../QUICKSTART_MCP.md) Â· [`docs/QUICKSTART_LLM_PROXY.md`](../QUICKSTART_LLM_PROXY.md). Architecture: [`docs/PROXY_ARCHITECTURE.md`](../PROXY_ARCHITECTURE.md).

### SDK hardening (compatibility tier)

The Python and TypeScript SDKs got the deepest pass since v0.4.0. The audit found that `GuardedTool`, `GuardedCrewTool`, and `GuardedPage` were silently bypassing the modern `invoke`/`ainvoke`/`stream`/`batch` API paths in LangChain, CrewAI, and browser-use respectively â€” `gt.func` and `gt._tool` were also one-line bypasses for any agent author. v0.5 fixes all of this:

- All three Python adapters rewritten as **composition wrappers with strict allowlists**. Every modern call path is gated. `gt.func` raises AttributeError with a security explanation.
- **`browser-use` adapter expanded** beyond `goto`. `GuardedPage` + `GuardedFrame` gate goto/click/fill/type/press/select/check/uncheck/evaluate/evaluate_handle/set_extra_http_headers/route/expose_function/add_init_script. Form values >256 chars auto-truncated to `<redacted; len=N>` for PII safety.
- **`data` policy scope** wired in for form-input gating. PII patterns enforced at policy evaluation time.
- **MCP Python adapter** â€” the `__main__` is no longer an empty server. Single-upstream proxying gateway via `--upstream "<cmd>"`. Per-frame error handling. Redactor applied to synthesized commands. (For multi-upstream namespaced gateways, use the new Go `agentguard-mcp-gateway` binary.)
- **Python â‰Ą 3.9** (drops 3.8). **Node â‰Ą 18** (TypeScript). CI matrix expanded.
- **Real-framework integration tests** in CI run weekly + per-PR against actual upstream packages.

### Architecture (unblocks v0.6 multi-tenant)

- **`PolicyProvider` interface** (`pkg/policy/provider.go`) â€” engines no longer load policies directly; they consume a provider. v0.6's database-backed multi-tenant provider drops in without engine changes.
- **`BufferedAsyncLogger`** (`pkg/audit/buffered.go`) â€” audit writes no longer block the request path. Bounded queue + N workers + disk-overflow recovery means `Log()` never blocks.
- **Tenant-aware URL routing** â€” `/v1/t/{tenant}/...` routes alongside the legacy `/v1/...` family. SDKs accept `tenant_id`/`tenantId` constructor parameter. v0.5 is single-tenant (`local`); v0.6 ships the multi-tenant surface.
- **Versioned wire schema** â€” `pkg/proxy/schema/v1/` with cross-language contract tests (Go â†” Python â†” TypeScript). `schema_version: "v1"` field on every request/response.
- **`agentguard check` CLI subcommand** â€” one-shot policy evaluation for CI pipelines. Four input modes (per-field flags, `--request '<json>'`, `--stdin`, `--batch` JSONL), four exit codes (0=allow, 1=deny, 2=approval, 3=error), severity-rank batch reducer.

### Code-correctness fixes (v0.4.x bugs that silently affected operators)

- **Rate-limit metrics no longer double-count.** Every operator's `agentguard_checks_total` and `agentguard_denied_total` was wrong proportional to the rate-limit hit rate.
- **`est_cost == 0` cumulative bypass closed.** Previously, a tool reporting est_cost=0 every action could accumulate real upstream LLM cost without tripping the session limit.
- **Per-agent override defaults to `merge`.** Previously, an override with `allow:` only silently dropped base policy denies (`rm -rf *` etc.) â€” silent privilege escalation. Explicit `override_mode: replace` opt-out reproduces v0.4.x behavior.
- **`time_window` without `require_prior` is a hard load error** (was a deprecation warning).
- **Glob `**` semantics** documented and locked by regression test (path-vs-domain asymmetry preserved as a stable contract).
- **`recoverPanic` middleware** on every HTTP handler + goroutine wrapper. Panic in any single handler no longer kills the process.
- **Audit log rotation wired by default.** `--audit-max-size-mb`, `--audit-max-backups`, `--audit-max-age-days`, `--audit-compress` flags.
- **Audit-log `bufio.Scanner.Err()` checked everywhere.** Oversized lines no longer silently truncate `/v1/audit` and the `require_prior` history adapter.

### Approval flow

- **`/v1/check` honors `approval_id` round-trip.** When the model retries a denied tool call after a human approves on the dashboard, the central server consults the approval queue and short-circuits with the resolved decision instead of producing a fresh approval cycle. The "approve once, model proceeds" UX works end-to-end.
- **Replay defense** â€” the round-trip validates the retry request matches the original on 7 operationally-meaningful fields (AgentID, Scope, Command, Path, Domain, URL, Action). An attacker who learns an approved id cannot replay it against a different action. New metric `agentguard_approval_replay_mismatch_total` for monitoring.

## Breaking changes

### Policy schema
- **`time_window` without `require_prior` now fails policy load.** Operators upgrading must remove time-window-only conditions from their policies. v0.4.1 emitted a deprecation warning; v0.5 promotes to a load error.
- **Per-agent override behavior** â€” defaults to `merge` (base deny rules inherit). To reproduce v0.4.x replace-all behavior, set `override_mode: replace` on the agent block.

### SDK
- **Python â‰Ą 3.9** required (3.8 dropped).
- **TypeScript** â€” Node â‰Ą 18.0.0 required.
- **`failMode` validation** â€” TS constructor throws on unknown values (was silently accepting).
- **Unknown kwargs** to `@guarded(**kwargs)` (Python) and `guarded(...)` (TS) now raise.
- **Adapter attribute access** â€” `GuardedTool.func`, `GuardedTool._tool`, and any non-allowlisted attribute now raise AttributeError. Code that depended on direct internal access via the wrapper must be updated to call gated methods.

### CLI
- The `agentguard` binary's `version` subcommand and the new `agentguard-mcp-gateway --version` / `agentguard-llm-proxy --version` all report `0.5.0`.

## Get started

```bash
# Install all three binaries
go install github.com/Caua-ferraz/AgentGuard/cmd/agentguard@v0.5.0
go install github.com/Caua-ferraz/AgentGuard/cmd/agentguard-mcp-gateway@v0.5.0
go install github.com/Caua-ferraz/AgentGuard/cmd/agentguard-llm-proxy@v0.5.0

# Start the central server
agentguard serve --policy configs/default.yaml --dashboard --api-key "$(openssl rand -hex 16)"

# Either: wire your MCP client (Claude Desktop / Cursor / Cline / Continue / Zed)
#   See docs/QUICKSTART_MCP.md and examples/<client>-config.json
#
# Or: point your OpenAI/Anthropic SDK at the LLM proxy
#   agentguard-llm-proxy --listen 127.0.0.1:8081 --policy configs/default.yaml \
#     --guard-url http://127.0.0.1:8080 --api-key "$AGENTGUARD_API_KEY"
#   export OPENAI_BASE_URL=http://127.0.0.1:8081/v1
#   See docs/QUICKSTART_LLM_PROXY.md and examples/<sdk>-config.py
```
